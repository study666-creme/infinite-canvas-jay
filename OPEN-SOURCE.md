# 开源与合规说明（卡藏提示词画布）

本项目是基于 AGPL-3.0 代码继续开发的「卡藏提示词画布」。产品 UI 可以使用自己的品牌、导航和视觉风格，但仓库层面的许可证与源码开放义务必须保留。

## 项目信息

| 项目 | 内容 |
| --- | --- |
| 当前产品名 | 卡藏提示词画布 |
| 当前公开源码 | https://github.com/study666-creme/infinite-canvas-jay |
| 上游项目 | https://github.com/basketikun/infinite-canvas |
| 协议 | AGPL-3.0，见 [LICENSE](LICENSE) |

## 可以做什么

- 可以改 UI、改品牌、改导航、改交互，把公开产品做成「卡藏」自己的体验。
- 可以在页面里移除显眼的上游 GitHub、版本弹窗、外部文档入口等非必要产品入口。
- 可以把卡藏资产、提示词、AI 生图/视频流程接入画布，作为自己的业务能力继续开发。
- 可以线上部署，但需要保持对应修改源码可获取，通常做法是保持当前 GitHub 仓库公开并与线上版本对应。

## 不能做什么

- 不要删除 `LICENSE`，也不要把 AGPL 代码改成闭源交付或闭源 SaaS。
- 不要在公开服务中隐藏源码获取方式后继续运营 AGPL 派生代码。
- 不要删除仓库级别的上游来源和许可证说明；这些可以放在文档里，不需要作为产品顶部按钮长期暴露。

## UI 与品牌边界

当前策略是：

- 产品首屏、顶部导航、画布内菜单使用「卡藏提示词画布 / 卡藏画布」品牌。
- 产品 UI 不再显示上游 GitHub 链接、版本检查入口、外部文档入口。
- 合规信息集中保留在仓库文档和许可证文件中，便于履行 AGPL 的来源与源码开放要求。

## 上线时的最低合规清单

- 保留 [LICENSE](LICENSE)。
- 保持当前公开源码仓库可访问，并确保部署版本的修改源码能在该仓库中找到。
- 如果产品页将来增加「关于 / 开源说明」页面，可链接到当前公开源码仓库，而不是上游仓库。
- 如果未来要闭源商用，需要先确认是否能获得上游商业授权或完全替换 AGPL 代码。
