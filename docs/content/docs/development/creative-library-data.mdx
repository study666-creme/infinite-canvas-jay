# 创作资料库数据

本地数据库位于 `web/data/creative-library.sqlite`，只用于摄取缓存、审核状态和失败重试。该流程是可选的私有扩展；公开浏览器 Agent 与 Vercel 部署不会读取 SQLite，也不会自动加载生成索引。

原始材料分别保留在：

```text
web/knowledge/creative/raw/          方法、笔记、官方文档和字幕
web/knowledge/creative/cases/raw/    完整故事与剧本案例
web/knowledge/creative/videos/inbox/ 视频收件箱
```

这些目录中的用户材料、Cookie 和 URL 清单默认不进入 Git。

## 数据表

- `library_sources`：来源、类型、内容哈希、摄取模式与状态。
- `knowledge_cards`：方法卡 JSON、分类、审核状态、可信度与来源权威度。
- `knowledge_card_sources`：知识卡与来源的多对多关系。
- `story_cases`：人物、关系、故事发动机、真实钩子、节奏和禁止照搬项，不保存完整原文。
- `ingest_jobs`：知识、案例和字幕任务的成功、失败与重试记录。

只有经过用户明确确认的 `verified` 知识卡和故事案例进入运行时检索。`auto_verified` 仅表示已经独立审核、仍待用户确认，并与普通 `candidate` 一样留在候选区；模型失败或证据不足的内容也不得进入运行时。

受控 Agent 蒸馏使用两阶段审核：候选代理逐来源生成草稿，独立审核代理再逐卡给出 `keep`、中文 `auditReason` 和可追溯 `evidenceSummary`。SRT/VTT 代表分块保留时间范围；证据摘要必须包含视频 ID 与时段。视觉依赖、低覆盖或严重 ASR 错词不能仅凭字幕通过审核。

来源权威度与召回作用域分开保存：`core` 用于通用创作骨架，`specialist` 按领域触发，`extension` 只在明确相关任务中触发，`corroboration` 只做术语和证据校核。完整书籍可产生多于五张卡；五张只是每轮创作召回的上限。

## 生成索引

- `creative-knowledge-pack.generated.ts`：审核后的方法卡。
- `creative-case-library.generated.ts`：审核后的故事案例结构。

这些文件默认不提交，公开构建也不包含。仓库内的本机 `canvas-agent` 可在私有说明或报告实际存在时按任务加载，并提供受 token 保护的命中预览；用户确认的项目事实和约束始终高于知识与案例。

## 常用命令

```bash
cd web
npm run knowledge:watch
npm run knowledge:status
npm run knowledge:ingest
npm run knowledge:cases
npm run knowledge:subtitle
```

格式、OCR、字幕、模型变量和版权边界见 `web/knowledge/creative/README.md`。
