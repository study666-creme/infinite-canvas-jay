# 创作总监 Agent

Infinite Canvas 是唯一面向用户的创作工作台。故事、活动参赛、知识蒸馏、剧本、资产、分镜、审核、预览、生成和返工都是同一个画布 Agent 的内部能力，不再维护一套并列的创作 Agent 产品。

画布 Agent 也不是一组互相独立、最后再拼接答案的聊天角色。它由总控维护一个结构化项目黑板，并只调用当前阶段真正需要的创作能力。网页模型、本机 Codex、Claude 或无界面 API 只是可替换执行后端，读取和更新的是同一份项目状态与画布成果。

本机 Codex 链路已经支持按画布创建或恢复对话、绑定工作目录、携带图片附件，并在用户确认工具调用后把结果写回画布。当前唯一必须与浏览器画布运行态强耦合的能力是 3D 导演台的载入、摆位、切镜和截图；其他创作推理通过项目状态、知识检索和通用节点工具协作。

创作形态先粗分为两种：AI 短剧是 `series`，负责分集、集间推进和跨集资产一致性；AI 短片是 `short_film`，只完成一个单片闭环，不强制前三集或连载钩子。活动/参赛是可覆盖两种形态的目标约束，精品叙事与快节奏是质量策略。

## 动态工作方式

1. 读取当前画布与用户输入。
2. 判断用户已经完成到哪一步，而不是强制从头跑固定流程。
3. 更新创作形态、当前阶段、完成度、活动约束、已确认常量、待确认问题和下一缺口。
4. 按需推进故事、剧本、资产、分镜、审核、预览、生成或返工。
5. 把成果写入可见节点，并记录版本、审核状态和用户确认状态。

参赛和活动需求属于同一创作链路的约束模式。平台、时长、画幅、交付格式、禁区和加分项会先写入项目黑板，再约束后续输出。

## 知识来源的真实边界

内置部分是原创的结构化方法总结，不包含完整原书、完整课程或逐字字幕。它覆盖故事、人物、台词、场景、导演、表演、摄影、分镜、色彩、剪辑、声音、AI 视频提示和发布质检等稳定原则。

用户可以在本地资料库加入自己有权使用的书摘、笔记、官方文档、字幕和完整案例。只有经过以下流程的内容才进入正式检索：

1. 读取与来源去重。
2. 提炼可执行原则、触发条件、证据摘要和适用边界。
3. 由隔离的第二轮模型检查材料是否真正支持结论。
4. 通过阈值后进入正式卡；不确定内容保留为候选。

Agent 每轮只检索与任务相关的少量方法卡。完整故事案例只在点子、人物、故事或剧本任务中使用，并明确记录“可借鉴结构”和“禁止照搬项”。

## 质量门槛

- 不把“电影感、高级感、网感”当作无意义万能词。
- 不采纳无法追溯的提示词玄学、短期平台漏洞和低质量套路集合。
- 分镜必须包含主体、动作、空间关系、镜头、时序变化、声音/台词要点和必要禁止项。
- 审核不仅给评价，还要产出可执行的二次修正版。
- 用户确认的角色、场景和活动硬约束始终高于外部知识与案例。

## 不是模型后训练

这套能力属于系统上下文、结构化状态和检索层，不会改变底层模型权重。它不能声称“已经读完所有书”，也不能保证自动掌握最新热榜。资料未成功摄取或未通过审核时，Agent 不应假装知道其具体内容。

本地资料库操作见仓库的 `web/knowledge/creative/README.md`。
