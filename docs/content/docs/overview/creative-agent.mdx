# 画布 Agent

画布 Agent 读取当前 Infinite Canvas 状态，并根据用户本轮指令调用真实工具。网页模型、本机 Codex、Claude 或无界面 API 都可以作为执行后端；画布节点与项目状态才是共享数据层。

公开运行时只包含通用工具协议，不内置故事结构、短剧流程、导演方法、行业知识或维护者的个人预设。Agent 不会因为画布中出现某类内容就擅自启动一套固定工作流。

## 工作方式

1. 先读取当前画布、选区和相关节点。
2. 只处理用户明确要求的目标；有实质歧义时先澄清。
3. 使用最小范围的节点、连线、生成或导演台工具完成操作。
4. 写入、删除、生成和其他有后果的动作遵循界面确认规则。
5. 根据真实工具结果汇报完成、失败或仍待确认的部分，不伪造执行结果。

Agent 可以按需维护结构化项目状态，例如阶段、约束、已确认结果和待办。这些字段用于跨轮次协作，不代表必须按某个行业顺序推进。

## 执行后端

网页 Agent 直接使用浏览器中的画布工具。本机 Agent 通过 `canvas-agent`、Codex app-server 或 Claude CLI 与 Infinite Canvas MCP 连接。

Codex 链路支持按画布创建或恢复对话、绑定工作目录、携带图片附件，并在用户确认工具调用后把结果写回画布。普通 Codex 代码任务不会自动获得画布提示或 MCP 上下文。

3D 导演台的载入、摆位、切镜和截图必须操作当前浏览器中的 Three.js 实例；其他节点操作不依赖导演台运行态。

## 私有知识扩展

仓库提供可选的本地资料摄取和审核脚本。原始材料、蒸馏报告、知识正文、来源清单和个人提示均被 Git 忽略，因此公开构建不携带这些内容。本机 `canvas-agent` 在本地说明或报告实际存在时才启用扩展：私有说明通过文件头的 `triggers` / `excludes` 限定范围，知识卡按卡片自身内容检索；普通 Codex 代码对话不注入画布上下文。

私有部署可以显式增加检索层，但应满足以下边界：

- 只处理有权使用的材料。
- 候选提炼与独立审核分离，未经用户确认的内容不进入正式索引。
- 只保存原创结构摘要与可追溯证据，不发布完整受版权保护材料。
- 用户确认的项目事实始终高于检索结果和模型推断。
- 每轮只从 `verified` 卡中召回 `0-5` 张，不为凑数打满；`auto_verified` 仍需用户确认。
- 可通过受 token 保护的私有上下文预览接口检查命中词、排序、来源和可信度。
- 私有说明使用 frontmatter 的 `triggers` 与可选 `excludes` 声明作用域，避免无关画布任务被个人预设污染。

这类检索属于运行时上下文，不会改变底层模型权重，也不等于模型已经读过或掌握了某本书、课程或视频。

## 安全边界

- 本机 Agent 默认只监听 `127.0.0.1`。
- Agent token 可以访问本机工作区和工具，必须视为密钥。
- 公网反向代理需要 HTTPS、访问控制、来源限制和固定强随机 token。
- 工具失败时必须显示真实错误，不能把计划或推断写成已完成操作。

本机接入细节见仓库根目录的 `CANVAS-AGENT.md`；可选资料工具见 `web/knowledge/creative/README.md`。
