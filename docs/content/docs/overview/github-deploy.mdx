# GitHub 与 Vercel 部署

本 fork 仓库：**https://github.com/study666-creme/infinite-canvas-jay**  
上游：**https://github.com/basketikun/infinite-canvas**

## 1. 克隆与本地开发

```bash
git clone https://github.com/study666-creme/infinite-canvas-jay.git
cd infinite-canvas-jay/web
npm install
npm run dev
```

浏览器打开 `http://localhost:3000`。

## 2. 推送到 GitHub（维护者）

```bash
git remote -v
# origin    → study666-creme/infinite-canvas-jay
# upstream  → basketikun/infinite-canvas

git push -u origin main
```

Fork 时请保留 [LICENSE](https://github.com/basketikun/infinite-canvas/blob/main/LICENSE) 与 README 中的上游说明，并保留前端页面上的原作者标识。

更完整的步骤、Docker / Render 与协议说明见仓库根目录 [DEPLOY.md](https://github.com/study666-creme/infinite-canvas-jay/blob/main/DEPLOY.md)。

## 3. Vercel 部署（推荐）

1. 打开 [Vercel](https://vercel.com/) → **Add New → Project** → 选择 `study666-creme/infinite-canvas-jay`。
2. **Root Directory** 设为 `web`（Vercel 项目设置里填，**不要**依赖仓库根目录的 `rootDirectory` 字段）。
3. Framework：**Next.js**；Build：`npm run build` 或 `bun run build`。
4. 部署完成后访问 `https://<项目名>.vercel.app`。

部署后无需在 Vercel 配置 AI Key：用户在浏览器右上角自行填写 Base URL 与 API Key。

### Prompt Hub 联调

1. 画布内 **设置 → Prompt Hub** → 用 [卡藏](https://prompt-hubs.com) 账号登录。
2. **插入素材 → Prompt Hub 卡片库**：从云端卡片库插入图片 + 提示词。
3. 图片节点 **右键 → 存为 Prompt Hub 卡片**。

Worker 已允许 `*.vercel.app` 跨域；自定义域名需在 Prompt Hub Worker `CORS_ORIGINS` 中追加。
