# 视频生成与播放

画布支持 OpenAI 风格异步视频、Seedance Agent Plan 和 xAI/Grok Imagine 视频任务。生成任务成功不等于浏览器一定能直接播放上游 URL，播放链路还取决于 CORS、鉴权、媒体格式和网关能力。

## 节点处理顺序

1. 创建异步任务并轮询状态。
2. 优先下载任务内容接口返回的真实媒体。
3. 必要时通过兼容网关代理远程 URL。
4. 校验 Blob 类型与文件魔数，拒绝 JSON/HTML 错误体。
5. 写入 IndexedDB 并保存 `storageKey`。
6. 播放时优先恢复本地 Blob，失败后再尝试远程来源。

## 网关接口

返回远程 CDN URL 的服务建议提供：

```http
GET /v1/media/fetch?url=<encoded-remote-url>
Authorization: Bearer <api-key>
```

响应要求：

- 2xx 状态。
- 视频二进制正文。
- 正确的 `Content-Type`，例如 `video/mp4`。
- 不把 JSON 错误对象包装成 200 视频响应。
- 对允许代理的目标域名、协议、重定向和响应大小设置限制，避免 SSRF 与资源滥用。

Seedance 网关还应支持任务内容接口：

```http
GET /v1/contents/generations/tasks/{id}/content
```

## 配置

画布 `Base URL` 指向真实兼容网关，例如：

```text
http://127.0.0.1:8000/v1
```

模型、时长、比例、清晰度、声音、水印和参考媒体支持程度由上游决定。Agent Plan 不一定实现 OpenAI `/models`，无法拉取列表时需要手动填写模型名。

## 排查

1. 在浏览器 Network 中确认创建任务、轮询和媒体请求的实际 URL 与状态码。
2. 下载失败时检查响应头和前几个字节，确认不是 JSON/HTML。
3. 确认网关进程已经重新构建并包含 `/v1/media/fetch` 与任务内容路由。
4. 节点重试会丢弃已识别的坏缓存；仍失败时清理对应 IndexedDB 媒体项。
5. 网关重启后任务内存丢失时，只有仍可访问的远程 URL 或持久化任务记录能够恢复旧节点。

## 安全

媒体代理是服务端网络请求入口，必须校验用户身份、目标 URL、响应大小、超时和重定向。不要把第三方 API Key 暴露到公开日志或 `NEXT_PUBLIC_*` 环境变量。

真实模型和网关验证状态见[待验证清单](/docs/progress/pending-test)。
